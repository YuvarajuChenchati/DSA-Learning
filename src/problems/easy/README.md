## ‚ö†Ô∏è Common Mistakes

### Not Handling Edge Cases
- **Problem**: Not checking for empty arrays, null inputs, or single elements
- **Solution**: Always check edge cases first
- **Example**: `if (nums == null || nums.length == 0) return result;`

### Off-by-One Errors
- **Problem**: Incorrect array bounds checking
- **Solution**: Always check `i < arr.length` not `i <= arr.length`
- **Example**: `for (int i = 0; i < arr.length; i++)` not `i <= arr.length`

### Wrong Data Structure
- **Problem**: Using wrong data structure for the problem
- **Solution**: Use HashMap for lookups, HashSet for membership, ArrayList for dynamic arrays
- **Example**: Use HashMap for Two Sum, not ArrayList

### Not Understanding Problem Requirements
- **Problem**: Not reading problem carefully
- **Solution**: Read problem statement thoroughly, understand constraints
- **Example**: Check if input is sorted, if duplicates are allowed

### Incorrect Time/Space Complexity
- **Problem**: Not analyzing complexity correctly
- **Solution**: Analyze both time and space complexity
- **Example**: Two Sum is O(n) time and O(n) space, not O(n¬≤)

### Not Testing Examples
- **Problem**: Not testing with provided examples
- **Solution**: Walk through examples before coding
- **Example**: Test with example inputs and verify output

### Not Considering Edge Cases in Logic
- **Problem**: Logic fails for edge cases
- **Solution**: Consider empty, single element, and boundary cases
- **Example**: Handle empty string, single character, array with one element

## üí° Problem Solving Tips

- **Start with brute force**: Get a working solution first
- **Optimize step by step**: Improve time/space complexity
- **Consider edge cases**: Empty inputs, single elements
- **Use appropriate data structures**: Choose the right tool
- **Practice regularly**: Consistency is key
